# ТЗ: Вкладка "Content" (Анализ текста страницы)

## 1. Назначение

Новая вкладка в расширении SERP Zen — анализ текстового контента страницы: статистика по тексту, частотный анализ слов/фраз, стоп-слова. Логика повторяет референс (см. приложенные скрины), реализуется как новый функционал расширения.

Вкладка состоит из трёх блоков:

1. **Content Analysis Report** (Text Statistics) — сводная таблица метрик
2. **Top Words** — частотный анализ слов и словосочетаний (1/2/3-словные)
3. **Stop Words** — список найденных стоп-слов с частотой

---

## 2. Источник анализируемого текста

### 2.1 Переключатель режима извлечения

В верхней части вкладки — переключатель (toggle/segmented control) с двумя режимами:

```
[ Вся страница ]   [ Основной контент ]
```

- **Вся страница** — берётся весь видимый текст `document.body` (с учётом исключений ниже)
- **Основной контент** — эвристическое извлечение только содержательной части страницы, без навигации/футера/сайдбаров

При переключении режима — весь анализ (все 3 блока) пересчитывается заново для текущей страницы без повторной выгрузки/перезагрузки.

### 2.2 Логика извлечения текста (общая для обоих режимов)

Извлекается только видимый текст:

- Исключаются элементы с `display: none`, `visibility: hidden`, `aria-hidden="true"`
- Исключаются `<script>`, `<style>`, `<noscript>`, `<template>`
- Текст из `alt`/`title` атрибутов **не** включается в основной подсчёт (отдельная метрика, опционально)
- Множественные пробелы/переводы строк нормализуются в один пробел между словами

### 2.3 Логика режима "Основной контент"

Эвристика приоритета (по аналогии с алгоритмами Readability.js / Mozilla Reader Mode):

1. Поиск семантических тегов: `<main>`, `<article>`
2. Если не найдены — поиск по распространённым селекторам: `[role="main"]`, `.content`, `.post`, `.entry-content`, `#content` и т.п.
3. Исключение блоков: `<nav>`, `<header>`, `<footer>`, `<aside>`, элементы с классами/ролями `nav`, `menu`, `sidebar`, `comments`, `cookie`, `banner`, `ad`/`advertisement`
4. Если эвристика не дала результата (нет ни одного однозначного контентного блока) — fallback на режим "Вся страница" + предупреждение в UI: "Не удалось определить основной контент, показан анализ всей страницы"

### 2.4 Кэширование и пересчёт

- Результат анализа держится в памяти панели на сессию (не персистится между перезагрузками страницы)
- Кнопка "Обновить анализ" — для SPA / динамически подгружаемого контента

## 3. Блок 1: Content Analysis Report (Text Statistics)

### 3.1 UI

Таблица в 3 колонки: **Parameter** (с иконкой `ⓘ` — tooltip с объяснением метрики) / **Value** / **Unit**. Полностью повторяет вид референсного скрина.

### 3.2 Состав метрик и формулы расчёта

| Параметр                      | Unit                                              | Формула / логика                                                                                                                                                                                                                                                                           |
| ----------------------------- | ------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Total Words                   | words                                             | Количество токенов после разбиения текста по границам слов (regex на основе Unicode word boundaries, поддержка кириллицы/латиницы)                                                                                                                                                         |
| Unique Words                  | words                                             | Количество уникальных слов после нормализации (lowercase; без учёта морфологии/лемматизации в MVP)                                                                                                                                                                                         |
| Percentage of Unique Words    | %                                                 | `Unique Words / Total Words * 100`, округление до 2 знаков                                                                                                                                                                                                                                 |
| Stop Words Count              | words                                             | Количество слов, входящих в объединённый список стоп-слов (см. раздел 5)                                                                                                                                                                                                                   |
| Percentage of Stop Words      | %                                                 | `Stop Words Count / Total Words * 100`                                                                                                                                                                                                                                                     |
| Water (redundancy)            | %                                                 | Метрика "водности" текста — доля слов, не несущих смысловой нагрузки. Обычная формула: `100 - (значимые слова / Total Words * 100)`, где значимые слова = слова за вычетом стоп-слов и общих "связующих" слов. Точную формулу нужно сверить с референс-инструментом (см. открытые вопросы) |
| Average Sentence Length       | words/sentence                                    | `Total Words / Sentence Count`. Разбиение на предложения — по `.`, `!`, `?`, с обработкой исключений (сокращения, числа с точкой, многоточия)                                                                                                                                              |
| Readability                   | оценка + текстовая метка (например "Fairly Easy") | Индекс Flesch Reading Ease, переведённый в текстовую шкалу (Very Easy / Easy / Fairly Easy / Standard / Fairly Difficult / Difficult / Very Confusing — по стандартной шкале Flesch)                                                                                                       |
| Characters (excluding spaces) | chars                                             | Количество символов в тексте без учёта пробелов/переводов строк                                                                                                                                                                                                                            |
| Flesch–Kincaid                | grade                                             | Формула Flesch–Kincaid Grade Level: `0.39 * (words/sentences) + 11.8 * (syllables/words) - 15.59`                                                                                                                                                                                          |
| Gunning Fog                   | index                                             | Формула Gunning Fog Index: `0.4 * [(words/sentences) + 100 * (complex words/words)]`, где complex words — слова с 3+ слогами                                                                                                                                                               |

> ⚠️ Подсчёт слогов для формул Flesch-Kincaid/Gunning Fog не тривиален для не-английских языков (для русского нет единого стандартного алгоритма слогоделения). Нужно зафиксировать: считаем эти метрики только для англоязычного текста, либо адаптируем слогоделение под кириллицу (подсчёт гласных как приближение) — см. открытые вопросы.

### 3.3 Валидация / edge-cases

| Ситуация                                                      | Поведение                                                                                                                                                                                                               |
| ------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| На странице нет текста (пустой body, чисто медийная страница) | Блок показывает пустое состояние: "Недостаточно текста для анализа" вместо таблицы с нулями                                                                                                                             |
| Очень короткий текст (< 50 слов)                              | Таблица отображается, но рядом предупреждение: "Текста недостаточно для точного расчёта читаемости" — метрики Flesch-Kincaid/Gunning Fog/Readability помечаются как низкая достоверность (иконка ⚠️ рядом со значением) |
| Текст на нескольких языках одновременно                       | Метрики считаются по всему тексту без разделения по языкам (ограничение MVP, см. открытые вопросы)                                                                                                                      |

## 4. Блок 2: Top Words

### 4.1 UI

Переключатель вкладок над таблицей: `[1-word] [2-words] [3-words]`. Таблица с прокруткой: **Phrase** / **Count** / **%**, отсортирована по убыванию Count. Полностью повторяет вид референсного скрина (включая подчёркнутые слова — видимо стилизация под ссылку/выделение, нужно уточнить у дизайнера, оставляем как есть).

### 4.2 Логика расчёта

- **1-word**: частотность отдельных слов (без исключения стоп-слов — на референс-скрине видно, что "для", "на", "в", "и" — стоп-слова, но они **показаны** в Top Words. Соответственно Top Words считает **все** слова, включая стоп-слова)
- **2-words / 3-words**: n-граммы (биграммы/триграммы) — последовательные сочетания слов в исходном порядке текста, без пропуска стоп-слов внутри сочетания
- `%` рассчитывается как `Count конкретной фразы / Total Words (для 1-word) или Total N-grams (для 2/3-words) * 100`
- Регистр нормализуется (lowercase) для подсчёта частоты, но это не должно объединять разные по смыслу слова (предлоги типа "Для"/"для" считаются одним словом — корректно)
- Сортировка — по убыванию Count; при равном Count — алфавитная сортировка (вторичный критерий, чтобы порядок был детерминированным)

### 4.3 Валидация / edge-cases

| Ситуация                                                                    | Поведение                                                                                                                                                                                                                                              |
| --------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Текста недостаточно для 3-словных сочетаний (например, текст короче 3 слов) | Вкладка "3-words" доступна, но таблица показывает пустое состояние "Недостаточно данных"                                                                                                                                                               |
| Знаки пунктуации внутри n-грамм                                             | Пунктуация не входит в состав фразы при подсчёте, но порядок слов в исходном тексте сохраняется (не схлопывать предложения через знак конца предложения в одну n-грамму — то есть биграмма не должна "перепрыгивать" через точку/восклицательный знак) |

## 5. Блок 3: Stop Words

### 5.1 UI

Таблица с прокруткой: **Word** / **Count** / **Percentage**. Полностью повторяет вид референсного скрина.

### 5.2 Источник списков стоп-слов

Объединённый (multi-language) список стоп-слов — **всегда применяются все языковые списки одновременно**, без автоопределения языка страницы и без ручного выбора пользователем. Минимальный набор языков на старте:

- Русский
- Английский
- Испанский
- Украинский

(Список языков расширяемый — см. раздел 7, открытые вопросы, по поводу источника словарей)

### 5.3 Логика

- Слово засчитывается как стоп-слово, если оно (в lowercase) входит в объединённый список любого из поддерживаемых языков
- Числа также считаются отдельной категорией "стоп-слов" по аналогии с референс-скрином (на скрине видно "404", "1012", "1320" — то есть числовые токены тоже попадают в этот список). Логика: если токен состоит только из цифр — он автоматически идёт в Stop Words с отдельной пометкой типа (тип токена: `word` / `number`, опционально — отдельная колонка или иконка-индикатор в таблице)
- `Percentage` = `Count конкретного стоп-слова / Total Words * 100`
- Сортировка — по убыванию Count

### 5.4 Валидация / edge-cases

| Ситуация                                                                    | Поведение                                                                                                                                             |
| --------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| Стоп-слов на странице не найдено вообще                                     | Пустое состояние: "Стоп-слова не найдены"                                                                                                             |
| Слово совпадает в разных языковых списках (например, омонимы между языками) | Засчитывается один раз как стоп-слово, источник языка не отображается (для MVP не критично, можно опционально показывать tooltip "найдено в: ru, en") |

## 6. Общие UI-требования

1. Единый визуальный стиль таблиц для всех трёх блоков (как на референс-скринах): серый заголовок таблицы, белые строки, тонкие разделители
2. Скроллируемые таблицы (Top Words, Stop Words) — фиксированная высота контейнера с вертикальным скроллом, видимым на скрине скроллбаром справа
3. Иконка `ⓘ` рядом с каждым параметром в Text Statistics — открывает tooltip/popover с кратким объяснением метрики и формулой (важно для не-технических пользователей расширения)
4. Состояние загрузки — пока идёт токенизация/подсчёт (может быть заметно на больших страницах) — skeleton-заглушки в таблицах, не блокирующие переключение вкладок Top Words (1/2/3-words)
5. Кнопка "Обновить анализ" — пересчёт без перезагрузки страницы (актуально для SPA с подгружаемым контентом)
6. Переключатель "Вся страница / Основной контент" должен быть всегда видим в верхней части вкладки Content, независимо от скролла (sticky), так как влияет на все 3 блока сразу

## 7. Открытые вопросы (требуют решения до разработки)

1. **Точная формула "Water (redundancy)"** — нужно сверить с конкретным алгоритмом референс-инструмента (на скрине пример: 2216 слов → 77.84% water при 1.62% стоп-слов — то есть формула явно не сводится к простому "100% - % значимых слов" в наивном виде; нужен реверс-инжиниринг логики или явное описание от заказчика/референса)
2. **Подсчёт слогов для Flesch-Kincaid / Gunning Fog на кириллице** — стандартного алгоритма нет. Варианты: (а) считать эти метрики только для текста, определённого как английский; (б) использовать приближение через подсчёт гласных букв как "слогов" для кириллицы — менее точно, но даёт хоть какой-то результат
3. **Состав мультиязычного списка стоп-слов** — зафиксировать точные списки на старте (например, использовать открытые датасеты типа `stopwords-iso` для нужных языков) и формат их хранения в коде расширения (статичный JSON-файл в бандле расширения)
4. **Лемматизация / нормализация словоформ** — в MVP не предусмотрена (разные падежи/формы одного слова считаются разными словами в Top Words и Unique Words). Подтвердить, что это ок для первой версии
5. **Определение типа токена "число" в Stop Words** — нужно ли также относить к "числам" смешанные токены типа "404" внутри текста типа версий/кодов ошибок (как на скрине — выглядит как контент о технической документации), или это просто общее правило "любой токен из цифр → число"
6. **Источник текста при наличии iframe** — учитывается ли текст внутри `<iframe>` (если same-origin) или анализ ограничен только основным документом
